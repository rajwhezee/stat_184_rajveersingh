## Repo Management Plan

### Goal
Maintain a clean, well-documented repo that demonstrates good version control practices.

### Needs
- Git and GitHub
- A main branch for stable work
- A dev branch for active development

### Steps
1. Keep main branch stable and only merge via pull requests.
2. Do all active work on the dev branch.
3. Write meaningful commit messages following "If applied, this commit will..." format.
4. Use GitHub Issues to track tasks and bugs.
5. Label issues appropriately and close them when resolved.
6. Open a pull request with a title and description when merging dev into main.
